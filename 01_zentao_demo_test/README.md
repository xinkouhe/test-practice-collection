# 禅道 Demo API 测试（用户模块）

本项目基于禅道 Demo 环境，设计并执行了一套 **RESTful API 用户模块测试用例**，覆盖 Token、用户查询、用户创建、用户修改、用户删除等核心流程。

---

## 📌 测试范围

| 模块 | 接口 | 覆盖点 |
|------|------|--------|
| Token | 获取 Token | 参数校验、状态码、响应结构 |
| User | 获取用户信息 | 权限、字段完整性 |
| User | 获取个人信息 | 字段结构、角色信息 |
| User | 创建用户 | 必填项、参数校验、错误码 |
| User | 修改用户 | 权限、字段更新 |
| User | 删除用户 | 权限、资源存在性 |
| User | 用户列表 | 分页、权限、字段结构 |

## 📌 主要发现的问题（缺陷总结）

### 1. 创建用户接口文档与实际行为不一致
- 文档未标注 `gender` 为必填  
- 实际返回：`『性别』不能为空`

### 2. 修改用户信息接口权限异常
- Demo 账号理论上具备管理员权限  
- 实际返回：`403 Access not allowed`

### 3. 删除用户接口权限异常
- UI 可删除  
- API 返回 403，不一致

### 4. 无 Token 访问用户列表仍返回 200
- 预期：401  
- 实际：成功返回完整用户列表  
- 可能是 APIfox 全局 Token 自动注入

### 5. POST 修改用户信息返回 200 但无内容
- 不符合 REST 规范  
- 应返回 405 或 4xx
