# 禅道 API 缺陷报告（用户模块）

本报告基于禅道 Demo API 测试结果整理，包含接口行为与文档不一致、权限异常、状态码异常等问题。

---

## 🐞 缺陷 1：创建用户接口文档与实际行为不一致
**接口**：POST /users  
**问题**：文档未标注 `gender` 为必填，但实际返回错误：  
```
『性别』不能为空
```
**影响**：文档与实现不一致，影响自动化测试与客户端开发  
**严重级别**：中

---

## 🐞 缺陷 2：修改用户信息接口权限异常
**接口**：PUT /users/{id}  
**问题**：Demo 账号理论上具备管理员权限，但实际返回错误：  
```
403 Access not allowed
```
**影响**：权限模型不清晰  
**严重级别**：高

---

## 🐞 缺陷 4：无 Token 访问用户列表仍返回 200
**接口**：GET /users  
**预期**：401  
**实际**：200 + 返回完整用户列表  
**影响**：严重安全问题  
**严重级别**：高  
**备注**：可能是 APIfox 自动注入 Token，但 API 本身未校验

---

## 🐞 缺陷 5：POST 修改用户信息返回 200 但无内容
**接口**：POST /users/{id}  
**预期**：405 或 4xx  
**实际**：200 + 空响应  
**影响**：不符合 REST 规范  
**严重级别**：中
